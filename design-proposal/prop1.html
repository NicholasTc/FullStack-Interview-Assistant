Backend Auth/Security Entry Level

How would you explain the difference between session-based authentication and JWT authentication?

Answer as if you are speaking to an interviewer. Start with the core difference, then mention storage, validation, tradeoffs, and when you might choose each approach.

After submission

Model Answer

Sessions store authentication state on the server and send a session id to the client. JWTs store signed claims in a token that the server can verify without a session lookup.

Key Checklist

  • Explains server-side state vs signed token claims.
  • Mentions cookies, storage, expiration, and refresh.
  • Names tradeoffs around revocation and scaling.

Self Score

Note To Remember

Always mention revocation: sessions are easier to invalidate immediately, while JWTs need expiration, blocklists, or token rotation.
